Design review & redesign proposal

mzPeak Viewer — Redesign Handoff

A structure-level redesign of the unified, capability-adaptive mzPeak viewer (the merge of mzPeakIV imaging + mzPeakExplorer) for proteomics and imaging-MS researchers. The brief: sharpen information hierarchy, make empty/loading/error states honest, raise accessibility, and make the imaging round-trip — m/z → ion image → pixel → spectrum — self-evident, all without leaving the dense, instrument-grade aesthetic or the existing token system.

Scope: 6 connected screens Fidelity: mid-fi wireframe Type: IBM Plex Sans / Mono Accent: #3b54da Client-side, static-deployable No new color tokens

01 What this redesign is — and isn’t

This is a wireframe-level review, not a reskin. The visual language is unchanged: hairline-bordered light chrome wrapping a dark “data stage”, IBM Plex Sans for chrome and IBM Plex Mono for every measured value, the OpenMS electric-blue #3b54da accent used sparingly, viridis/inferno as the only data colour. Every change below is a layout, copy, state, or affordance move that can be implemented against the current component set. Nothing asks for a new palette, a heavier icon family, or decorative gradients.

  • Kept: the 2px spacing scale, 28px controls, the dark --ink stage with dot-grid, percentile clip + colormap + log scale, the real role=tablist sidebar, route=false pixel-pick into an in-place dock, the parquet footer inspector with on-demand sampling.
  • Changed: information hierarchy on every view, the empty/loading/error copy and behaviour, the discoverability scaffolding around the imaging loop, and the accessibility surface (labels, focus, live regions, keyboard pixel-picking).

The central thesis: make the round-trip a guided loop

The product’s reason to exist is one correct round-trip. Today a first-time user can open an imaging file and never realise it’s possible. The redesign threads a single, repeated motif — a numbered loop — from the start page into the nav into the imaging view itself.

① Pick m/z + tolerance→ ② Render ion image→ ③ Click a pixel→ ④ Read its spectrum↺

02 Screen-by-screen changes

1

Start / file-open

The front door. It must explain the product, earn trust (nothing is uploaded), and offer three open paths without ambiguity.

BeforeLogos, headline and paragraph at equal weight; three visually identical demo tiles; two equal grey “open mode” buttons; a spinner-only loading state; terse one-line errors.
AfterHeadline → value prop → privacy line stacked by priority; the imaging demo is highlighted with a ↻ round-trip caption; primary “Open from cloud” + quiet “Download & open”; a determinate, cancellable MB-of-MB load; specific errors that name the file and the expected format.
  • Drop-zone gains a sub-line naming the .mzpeak format (ZIP of Parquet + mzpeak_index.json) and an accepted-formats link — fewer wrong-file dead-ends.
  • Drag-over is an unmistakable accent fill with the file name echoed and a “Release to open” confirmation.
  • A11y: drop-zone is role=button with a 3px focus ring and Enter/Space; the URL field gets a real label.
2

Capability-adaptive shell

The persistent chrome whose left nav grows an Imaging (MSI) group only when the file carries spatial coordinates, and a Chromatograms entry only when chromatograms exist.

BeforeGroups appear or vanish silently; the round-trip order isn’t encoded in the nav; capability state is buried in Summary; MiniStats is a loose key/value stack.
AfterA quiet caption on the group header (“IMS columns ✓”) explains why a group is present; the first two MSI items are numbered ①②; capability badges sit in the top bar; MiniStats becomes a pinned file card.
  • The existing role=tablist + roving tabindex + Arrow/Home/End behaviour is retained, including auto-expanding the accordion that owns the active tab so focus is never trapped.
  • A one-line banner states what was detected and which groups are therefore shown/hidden — adaptivity becomes legible, not magical.
  • Non-imaging files degrade gracefully: the MSI group is withheld and Chromatograms is surfaced in its place.
3

Summary overview

The orientation page after open: who is this file, what can it do, where do I go next.

BeforeFour metric tiles lead with no file identity; capabilities are a long mixed StatRow list; the TIC thumbnail is decorative; no explicit next step.
AfterAn identity line anchors the page above its vitals; capabilities become a scannable badge grid with detection signals + confidence; the TIC thumbnail becomes an “Open ion-image explorer →” button; three explicit next-step actions hand off to the right view.
  • Unsupported features surface as amber chips with tooltips, so results are never silently incomplete.
  • Raw detail (file rows, archive members) is demoted into disclosures to keep the hero uncluttered.
4

Spectra view

m/z vs intensity for the selected spectrum, with MS-level filtering and isobaric (TMT/iTRAQ) reporter-ion reading.

BeforeA long select + bare index input lead; centroid/profile only appears in a meta string; the zoom gesture is undocumented; TMT reporters are anonymous sticks; MS-level hides in a dropdown.
AfterPrev/Next + “#318 / 1684 in level” lead; a representation badge and a persistent “scroll to zoom · double-click to reset” hint sit by the plot; reporter ions become colour-matched channel pills (label · % of strongest · sample); MS level becomes a segmented control of only the levels present.
  • Axes are explicitly labelled (intensity / m/z (Th)) and the plot carries an aria-described summary.
  • Detection is honest: “9 / 10 channels detected (±5 mDa)”, undetected channels dimmed to a dash — never fabricated.
  • A one-line note exposes the cross-view link: picking a pixel in Imaging fills this exact plot.
5

Imaging view — the round-trip

The product’s payoff surface, and the screen the brief weights most heavily.

BeforeBare m/z/tolerance inputs with no units or map of what they produce; a blank dark stage; an unframed render wait; a hover readout as muted text far below the canvas; a legend floating loose; pixel-pick that can route away to Spectra.
AfterA 4-step stepper across the top lights as you progress; a labelled m/z control with Th/Da unit chips + example; a “warming…/ready · prefetched” badge for the background prefetch; a determinate render bar with pixel count; an on-stage readout chip + selection ring; a legend bound flush to the stage; and an in-place spectrum dock that keeps the ion image in view.
  • Keyboard pixel-picking (arrows move a cursor cell, Enter picks) plus an aria-live readout make the core interaction operable without a mouse.
  • The dock pre-announces step ④ even before a pixel is picked (“click any pixel above to load its spectrum here”), and its header carries full provenance: pixel x · y · #index · point count · centroid/profile.
  • The empty state coaches the exact next action rather than reading as broken.
6

Parquet structure inspector

The format-orientation surface — archive members and per-column parquet footers — for format implementers and curious scientists.

BeforeMembers and footers share one scroll; the manifest looks like any row; size/share are numbers to compare by eye; jargon with no bridge to the science; histograms would stall if automatic.
AfterA stable two-pane inspector keeps the selected member highlighted; the manifest is pinned with a badge and a “View JSON →” cross-link to Metadata; inline share bars make dominant arrays legible at a glance; an empty-state line frames columns as stored data arrays; the ≤50k-row sampled histogram stays opt-in.
  • Semantic table headers + expandable rows-as-buttons + tabular figures so values compare cleanly down each column.

04 Cross-cutting rationale

Accessibility

  • Real tablist semantics + roving tabindex retained; focus never trapped in a collapsed accordion.
  • Keyboard pixel-picking on the ion canvas with an aria-live readout.
  • Every control labelled; plots carry a described summary.
  • 3px navy focus ring (--focus-ring) honoured on all interactive chrome; contrast checked on the active-tab accent.

States

  • Empty states coach the next action and distinguish causes (not-loaded vs truly-empty).
  • Loading is determinate and cancellable wherever a byte count exists.
  • Errors name the artefact and the recovery path.
  • Missing data reads as an em-dash or a plain statement — never a fabricated value.

Density & hierarchy

The app is intentionally dense; the redesign tunes that density rather than loosening it. Primary actions and current position are raised; raw diagnostics are demoted into disclosures; tabular figures keep numeric columns aligned. The result reads as a more confident instrument, not a busier one.

05 Open questions for the team

  • Should the round-trip stepper persist across all MSI sub-views, or only on Ion image?
  • Is the background prefetch of common m/z windows already wired, or does the “warming” badge describe future behaviour?
  • Confirm the canonical unit label for m/z in the UI — Th vs m/z vs Da — and standardise.
  • Do we want the capability badges in the top bar on every view, or only until the user has visited Summary once?
