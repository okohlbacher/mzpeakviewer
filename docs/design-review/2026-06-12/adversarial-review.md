# Adversarial review — mzPeak Viewer redesign — 2026-06-12

## Summary

**29 consolidated findings: 7 Blocker · 14 Major · 6 Minor · 2 Info.**
Adversaries run: **codex** (ran, 26 raw findings), **vibe** (ran, ~24 raw findings — output noisy but table usable), **internal ×3** (feasibility, consistency/a11y, hallucination). All five ran successfully.

**Headline.** The generated prototype is a **strong layout/visual direction** that honors the structural hard-constraints — the three demo cards, dropzone, URL field, top-bar logo, capability-gated left nav with active-accent rail, dark imaging stage, colormap legend, persistent spectrum dock, and Share-copy behavior are all present and well-composed. But **as an implementation handoff it is a self-contained mock**: it never touches the real `@mzpeak/core` engine, it **fabricates dataset metadata** (inflated ×1000+), and it **invents engine capabilities** the reader cannot back. Its value is the *design*, not the *code* — the roadmap must port the layout and discard the fabrications.

Every Blocker/Major below was independently raised by ≥2 adversaries and/or verified directly against the repo (grep / file:line). The prototype's self-declared "DS runtime shim" deviation is **prototype-only and irrelevant** to the real Vite + `@mzpeak/ui-kit` build — disregard it.

## Adversary agreement

Convergence was unusually high. F-01 (fabricated fixtures), F-05/F-06 (mock engine + view-id mismatch), F-08/F-09 (invented controls + base-peak), F-12 (false "no hard-coded colours"), F-16 (TIFF→PNG), and F-04/F-13 (a11y) were each raised independently by 3–5 adversaries. No material disagreements; where only one adversary raised a point (e.g. F-03 sentinel, F-18 plot library, F-20 contrast) it was verified directly against the repo or CSS. codex and vibe both independently re-derived the demo-fixture, logo, view-id, and invented-control clusters from the repo without seeing the internal reports.

## What was NOT reviewed

- **No runtime/interaction testing** of the prototype (the present-mode demo cards did not navigate under synthetic clicks; views were assessed from source + the generator's own QA screenshots).
- **No pixel-level visual diffing** against the current app.
- **Contrast ratios were reasoned, not instrumented** (F-20 marked "measure").
- External reviewers ran **read-only**; their Blocker/Major rows were spot-checked against the repo before consolidation.
